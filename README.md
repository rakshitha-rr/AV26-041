# 🌾 AgriIntel: Smart Agriculture Intelligence System

AgriIntel is a comprehensive, AI-powered agricultural management platform designed to empower farmers in India (specifically Karnataka) with data-backed insights, ML-driven yield predictions, and proactive risk alerts. 

This project consists of a **Next.js** frontend dashboard and a highly modular **FastAPI** backend that utilizes Machine Learning, Natural Language Processing (NLP), and secure data persistence.

---

## ✨ Core Features

### 1. Machine Learning Yield Prediction
- **Engine:** `RandomForestRegressor` trained on synthetic datasets mirroring Karnataka's climate and soil patterns.
- **Inputs:** Rainfall, Temperature, Soil pH, Soil Moisture, Fertilizer, and Irrigation.
- **Output:** Predicted crop yield (tons/hectare), categorized performance (e.g., "Good", "Excellent"), and actionable farming recommendations.

### 2. What-If Simulator
- Allows farmers to simulate changes (e.g., increasing irrigation by 20%) and directly compare the projected yield against their current baseline conditions.

### 3. Proactive Risk Detection
- Rule-based engine analyzing real-time inputs to detect and warn against:
  - Drought Risk
  - Crop Stress
  - Disease Probability
  - Flood / Nutrient Deficiency

### 4. Stateful Persistence & Security (Databases)
- **Redis Session Management:** Farmers securely log in using their 12-digit Aadhaar number and phone number. A secure UUID session token is generated, stored in Redis (24-hour TTL), and used to protect API routes.
- **MongoDB History Tracking:** Automatically saves all yield predictions and risk alerts generated by logged-in farmers, allowing them to review their historical data securely.

### 5. Multilingual NLP Interface
- Complete translation and interpretation layer supporting **English (`en`)**, **Hindi (`hi`)**, and **Kannada (`kn`)**.
- Backend ML numerical outputs are intelligently mapped into translated, farmer-friendly natural language sentences.
- Frontend UI allows seamless dynamic language switching.

### 6. Communication & Offline Accessibility
- **Twilio SMS Webhook:** Farmers can interact with the system via SMS or WhatsApp without internet access.
- **Offline Query Parsing:** Parses raw SMS commands (e.g., `YIELD 750 28 6.5 45 80 60`) and returns ML predictions via text.
- **NLP Chat:** Intent detection for basic natural language queries (e.g., "What schemes am I eligible for?").

### 7. Voice AI Assistant (New)
- **Voice Assistant:** Farmers can speak their questions and receive spoken responses in their preferred language.
- **Features:** 
  - **Speech-to-Text (STT):** Converts spoken queries into text for processing.
  - **Text-to-Speech (TTS):** Converts AI responses back into natural-sounding audio.
  - **Multilingual:** Supports English, Hindi, and Kannada voice interactions.
  - **Integrated Pipeline:** A single endpoint handles the full audio-in, audio-out conversational loop.

### 8. Supplementary Modules
- **Government Schemes:** Real-time eligibility checker for schemes like PM-KISAN, PMFBY, and Soil Health Cards.
- **Soil Analysis:** Quality assessment and crop recommendations.
- **Yield Heatmap:** Mocked geographic visualization of crop yield performance across 20 districts in Karnataka.

## 🚀 Getting Started

### Prerequisites
- Node.js & npm (for Frontend)
- Python 3.9+ (for Backend)
- Redis server running locally (Port `6379`)
- MongoDB server running locally (Port `27017`)

### Environment Setup
Create a `.env` file in the `backend/` directory:
```env
# Server
HOST=0.0.0.0
PORT=8000
DEBUG=true

# Databases
REDIS_URL=redis://localhost:6379/0
MONGODB_URL=mongodb://localhost:27017
MONGODB_DB_NAME=agriintel

# Twilio (Optional, for SMS integration)
TWILIO_ACCOUNT_SID=your_sid
TWILIO_AUTH_TOKEN=your_token
TWILIO_PHONE_NUMBER=your_number
```

### 1. Start the Backend
```bash
cd backend
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
*API Documentation will be available at `http://localhost:8000/docs`.*

### 2. Start the Frontend
```bash
cd frontend
npm install
npm run dev
```
*Web application will be available at `http://localhost:3000`.*

## 🔐 API Security Flow
1. **Login:** Farmer sends POST to `/api/login` with `aadhaar_number` and `phone_number`. 
2. **Session Creation:** Backend verifies, generates a secure UUID token, stores it in Redis for 24 hours, and returns it.
3. **Protected Access:** Frontend passes token in `Authorization: Bearer <token>` header to protected routes (`/predict-yield`, `/history`, etc).
4. **Data Persistence:** Authorized actions are saved to MongoDB under the farmer's Aadhaar identifier.
5. **Logout:** Calling `/api/logout` deletes the session from Redis.
